🚀 SAYAN DEVOPS PROJECT

Jenkins CI/CD Pipeline | AWS EC2 | Docker | GitHub

📌 Project Objective

To build a complete CI/CD pipeline using Jenkins Master-Agent setup on AWS EC2. Automate code build, Docker image creation, push to Docker Hub and deployment.

🏗️ Architecture

GitHub → Jenkins Master → Jenkins Agent → Docker → Docker Hub → Deployment

🛠️ Tools Used

  • AWS EC2 (Free Tier)
  • Jenkins (Master-Agent)
  • Docker
  • GitHub
  • Ubuntu Linux

⚙️ Step-by-Step Setup

1. EC2 Setup

Create 2 EC2 instances:
- Jenkins Master
- Jenkins Agent

Open Ports:
22 (SSH)
8080 (Jenkins)
8081 (App)

2. Install Jenkins

sudo apt update
sudo apt install openjdk-17-jdk -y
sudo apt install jenkins -y
sudo systemctl start jenkins

3. Install Docker

sudo apt install docker.io -y
sudo usermod -aG docker ubuntu
sudo usermod -aG docker jenkins

4. Setup Agent

Add node in Jenkins
Connect using SSH
Add PEM key in credentials

📄 Dockerfile

FROM nginx:latest
COPY . /usr/share/nginx/html

⚡ Jenkins Pipeline

pipeline {
    agent { label 'agent-1' }

    environment {
        IMAGE = "raysayan/my-node-app"
    }

    stages {

        stage('Clone') {
            steps {
                git branch: 'main',
                url: 'https://github.com/ray-sayan/static-website-demo.git'
            }
        }

        stage('Build') {
            steps {
                sh 'docker build -t $IMAGE .'
            }
        }

        stage('Push') {
            steps {
                sh 'docker push $IMAGE'
            }
        }

        stage('Deploy') {
            steps {
                sh 'docker run -d -p 8081:80 $IMAGE'
            }
        }
    }
}

⚠️ Problems Faced & Fixes

1. Agent Not Connected

Fixed by adding correct PEM key.

2. Known Hosts Error

ssh-keyscan -H  >> ~/.ssh/known_hosts

3. Permission Denied

Fixed by adding SSH key in Jenkins credentials.

4. Git Branch Issue

Changed from master to main.

5. Docker Push Error

Fixed by using correct Docker credentials/token.
