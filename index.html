🚀 SAYAN DEVOPS PROJECT

Jenkins CI/CD Pipeline | AWS EC2 | Docker | GitHub

📌 Project Objective

Build a complete CI/CD pipeline using Jenkins Master-Agent setup on AWS EC2. Automate the process of fetching code from GitHub, building Docker images, pushing them to Docker Hub, and deploying the application.

🏗️ Architecture

Developer → GitHub → Jenkins Master → Jenkins Agent → Docker Build
→ Docker Hub → EC2 Deployment → Browser Access

🛠️ Tools & Technologies

  • AWS EC2 (minimum 4GB)
  • Jenkins (Master-Agent)
  • Docker
  • GitHub
  • Linux (Ubuntu)

☁️ EC2 Setup

Create 2 instances:
1. Jenkins Master
2. Jenkins Agent

Open Ports:
22 → SSH
8080 → Jenkins
8081 → App

⚙️ Jenkins Installation

sudo apt update
sudo apt install openjdk-17-jdk -y

sudo apt install jenkins -y
sudo systemctl start jenkins
sudo systemctl enable jenkins

🐳 Docker Setup

sudo apt install docker.io -y

sudo usermod -aG docker ubuntu
sudo usermod -aG docker jenkins

🔗 Jenkins Agent Setup

1. Add Node in Jenkins
2. Use SSH connection
3. Add PEM key in credentials

Common Issue:
Permission denied (publickey)

Solution:
Add correct private key in Jenkins credentials

📦 GitHub Integration

git clone https://github.com/ray-sayan/static-website-demo.git

IMPORTANT:
Use branch = main (not master)

📄 Dockerfile

FROM nginx:latest
COPY . /usr/share/nginx/html

⚡ Jenkins Pipeline

pipeline {
    agent { label 'agent-1' }

    environment {
        IMAGE = "raysayan/my-node-app"
    }

    stages {

        stage('Clone') {
            steps {
                git branch: 'main',
                url: 'https://github.com/ray-sayan/static-website-demo.git'
            }
        }

        stage('Build') {
            steps {
                sh 'docker build -t $IMAGE .'
            }
        }

        stage('Login') {
            steps {
                withCredentials([usernamePassword(
                    credentialsId: 'dockerhub-creds',
                    usernameVariable: 'USER',
                    passwordVariable: 'PASS'
                )]) {
                    sh 'echo $PASS | docker login -u $USER --password-stdin'
                }
            }
        }

        stage('Push') {
            steps {
                sh 'docker push $IMAGE'
            }
        }

        stage('Deploy') {
            steps {
                sh '''
                docker stop static || true
                docker rm static || true
                docker run -d -p 8081:80 --name static $IMAGE
                '''
            }
        }
    }
}

⚠️ Problems Faced & Solutions

1. Jenkins Agent Not Connecting

Error: Permission denied (publickey)

Fix:
Add PEM key in Jenkins credentials

2. Known Hosts Error

ssh-keyscan -H  >> ~/.ssh/known_hosts

3. Git Clone Failed

Error: Couldn't find revision

Fix:
Use branch: main

4. Docker Permission Error

sudo usermod -aG docker jenkins
sudo reboot

5. Docker Push Failed

Error: insufficient scope

Fix:
Use correct Docker token/password
